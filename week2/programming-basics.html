Programming Basics

Comments

Comments are extremely important in programming. A comment is a piece of code that is ignored by the language but can be read by humans. Comments make it easy for people reading your code to understand what is going on. It's good practice to write good, concise comments!

The JavaScript language has two types of comments:

  • Single line comments - start with // and finish at the end of the line.
// a short comment on a single line
  • Multi-line comments - start with /*, usually span multiple lines and end with */
/* This here is a multi-line
comment. It's called multi-line
because it can span multiple lines
*/

JavaScript Grammar

JavaScript adopts a C-style syntax because of its similarities with the C programming language.

A JavaScript program is made up of a series of statements that end with a new line or semicolon.

Here's an example of two statements, one on each line:

const name = "Elvis Duru"
alert(name)

This example can also be written using semicolons to end each statement as follows: 

const name = 'Elvis Duru';alert(name);

JavaScript lets you end statements without a semicolon because the JavaScript interpreter uses a process called Automatic Semicolon Insertion (ASI). This process will attempt to place semicolons at the end of lines for you. However, it's advisable to combine the two - write each statement on a newline and end them with semicolons to avoid syntax errors, or code minifiers and validators not working properly. 

const name = 'Elvis Duru';
alert(name);

A block is a group of related statements collected inside curly braces: 

{
	// a javascript block containing 2 statements
	const name = 'Elvis Duru';
	alert(name);
}

Blocks do not need to be terminated by a semicolon.

Whitespace (such as spaces, tabs and new lines) is used to separate different parts of each statement. It's good practice to use enough whitespace to format your code so it's neat and readable. Examples of this includes using white space to indent nested code and multiple lines to separate blocks of code.

Reserved Words in JavaScript

The JavaScript language has the following words reserved for use. This means that you cannot used them to name variables (or function parameters and object properties). 

abstract, await, boolean, break, byte, case, catch, char, class, const, continue, debugger, default, delete, do, double, else, enum, export, extends, false, final, finally, float, for, function, goto, if, implements, import, in instanceof, int, interface, let, long, native, new, null, package, private, protected, public, return, short, static, super, switch, synchronized, this, throw, throws, transient, true, try, typeof, var, volatile, void, while, with, yield

The following were removed from the ECMAScript 5/6 standard:

Primitive Data Types

There are seven different data types in JavaScript. Six of them are primitive data types:

  • String
  • Symbol (introduced in ES6)
  • Number
  • Boolean
  • Undefined
  • Null

Any value that is not one of the primitive data types listed above is an object. This includes arrays, functions and object literals.

You can find out the type of value using a special operator called type of. 

typeof 'hello'
<< 'string'
typeof 10
<< 'number'
typeof true
<< 'boolean'
typeof { ninja: 'turtle' }
<< 'object'
typeof [ 1, 2, 3 ]
<< 'object' (All non-primitive data types return a type of 'object'.)

Variables

A variable refers to a value stored in memory.

Declaring & Assigning

Variables must be declared before they can be used. ES6 introduced a new way to declare variables using the const and let keywords. const is used when the variable will not be reassigned to another value. let is used if the variable will be reassigned later in the program.

To assign a value to a variable, we use the = operator.

The following examples show you how to declare a variable and assign a value to it: 

/*
	This constant now has a value of the string literal 'Elvis'. 
	It won't be assigned to another value.
*/
const name = 'Elvis';

/*
	This variable now has a value of an integer 25.
	This value may change during the program.
*/
let age = 25;


// Meanwhile...Later in the program
age = 30; // This is acceptible if you know what you're doing ✅
name = 'Daniel' // You'll get an error!! ❌

If the variable references a non-primitive data type, such as an array, function or object, then using const will not make it immutable. This means that the underlying data inside the object can change or be mutated: 

const user = {name: 'Elvis', age: 25}; // an object representing a user
console.log(user.name);
<< 'Elvis'

user.name = 'Daniel'; // This works!

console.log(user.name);
<< 'Daniel'

Even though its possible to mutate an object declared using a const variable, you however cannot assign the variable to another object: 

// declare object
const user = {name: 'Elvis', age: 25};

// Attempt to reassign to another object
user = {name: 'Daniel', age: 30}
<< TypeError: Assignment to constant variable

You can declare and assign multiple variables at the once if you separate them with commas: 

let a = 1, b = 2, c = 3;

The var keyword

Previous to ES6, variables were declared using the var keyword. 

var name = 'Elvis';

This worked just like let. Although, the difference was that variables declared using let and const have block scope as discussed below.

Scope

As mentioned earlier, variables declared using const and let are block scoped, this means that their values only exist in the block they are declared in.

Lexical Scoping is a very important concept in JavaScript programming as well as some other languages. It refers to where a constant or variable is accessible by the program. There are two types of scopes in JavaScript, the global scope and the local scope.

Global Scope

Any variable declared outside a block is said to have a global scope. This means that it's accessible everywhere in the program.

Local Scope

Variables created inside blocks with let or const have a local scope. This means that such variable is only accessible within that block. If not declared using let or const it will be available globally. This can be demonstrated in the following example: 

const name = "Elvis";
{
	const name = "Daniel";
	age = 25;
}

console.log(name); // "Elvis"
console.log(age); // 25

Note that when you reassign a global variable to a new value inside a block, it will update the global variable: 

const name = "Elvis";
{
	name = "Daniel"; // The value of name is updated here
	age = 25;
}

console.log(name); // "Daniel"
console.log(age); // 25

Also when you try to reference a variable that was declared inside a block from outside, you get an error: 

{
	name = 'Elvis';
	age = 25;
}

console.log(name); // ReferenceError: name is not defined.

Naming Constants & Variables

When naming constants and variables, it makes sense to give them sensible names that describe what the variable represents; 

// Good examples
let pet = "Dog"
let age = 25;

// Bad examples
let p = "Dog"
let a = 25;

Constant and variable names can start with any upper or lower-case letter, an underscore _ , or dollar character, $ . They can also contain numbers, but cannot start with them. Constant and variable names are case sensitive, so username is different to USERNAME and userName.

The following are valid examples: 

$username
_age
firstName
last_name
address
SPEED_OF_LIGHT

Direct Assignment & Assignment By Reference

When you assign a primitive value to a variable, any changes you make are made directly to that value. 

const a = 1;
let b = a; // a = 1, b = 1

b = 2; // a = 1, b = 2

In the above example, a references the primitive number 1. We then assign b to the same value as a. At this point a and b both have the same value of 1. Then we reassign the value of 2 to b, but a still retains the value of 1.

This case is different for non-primitive values though, as this is done by reference, so any changes that are subsequently made will affect all references to that object: 

const userA = {age: 25};
let userB = userA; // userA.age = 25, userB.age = 25
userB.age = 30; // userA.age = 30, userB.age = 30

In the above example, the change to the age property of userA also results in the age property of userB changing as well. This is because both variables are referencing the same object, so any changes made to one of them affects the other.

Strings

A string is a collection of characters, such as letters and symbols. We can create a string literal by writing a group of characters inside quote marks: 

'Hello World'

You can also create a string using the inbuilt String constructor function: 

let str = new String("Hello World");
console.log(str); // "Hello World";

This method of creating strings is rare, it's preferable to use the string literal notation, not to mention that it requires less typing to use literals.

If you want to use double quote marks inside a string literal, you need to wrap the entire string with single quote marks. And if you want to use an apostrophe in a string, you will need to use the double quote marks to enclose the string, to prevent an error. Another option is to escape the quotation mark using the backslash character before the apostrophe.

'It's me Mario' // causes an error ❌
"It's me Mario" // this works ✅
'It\'s me Mario' // this also works ✅

Backslashes

The backslash is used to escape special characters in strings such as:

  • Single quote marks \'
  • Double quote marks \"
  • End of line \n
  • Carriage return \r
  • Tab \t

If you want to actually write a backslash, you need to escape it with another backslash: 

"This is a backslash \\"

Template Literals

Template literals are a special types of string introduced in ES6. Template literals begin and end with a backtick character `, as shown below: 

`Hello World`
`Hello, It's me Mario!` // Can contain quotes

// Can contain line breaks, which are all converted to a Line Feed (\n)
`This is the start...

....and this is the end`

They also allow interpolation of JavaScript code. This means that a JavaScript expression can be inserted inside a template string and the result will be displayed: 

let age = 25, name = "Elvis";
console.log(`${name} is ${age + 3} years old in three years!`; 
// Elvis is 28 years old in three years

Symbols

Symbols were introduced as a new primitive value in ES6. They are used to create unique values, which help to avoid any name collisions.

Symbols are the only primitives that don't have a literal form. The only way to create them is to use the Symbol() function. 

const uniqueID = Symbol();

I

It is recommended to add a description of the symbol inside the parentheses:

const uniqueID = Symbol('this is a unique ID');

Because symbols are primitive values, the typeof operator should return a type of 'symbol': 

typeof uniqueID; // 'symbol'

The description acts as a string representation of the symbol and is used to log the symbol in the console, making it useful for debugging purposes:

console.log(uniqueID); // Symbol(this is a unique ID)

You can manually access the description using the String() function:

String(uniqueID) // 'Symbol(this is a unique ID)'

It is possible for two variables to point to the same symbol if the for() method is used when the symbol is created:

/*
The variables A and B now both point to thesame symbolin memory. In this case the description 'shared symbol' also acts as a shared identifier 
for the symbol.
*/
const A = Symbol.for('shared symbol');
const B = Symbol.for('shared symbol');

The main use-case for symbols is as object property keys. The uniqueness of symbols, mean that it’s impossible for the names of any properties to clash with each other if they are symbols.

Numbers

Numbers can be integers (whole numbers, such as 5) or floating point numbers (often referred to as just 'decimals' or 'floats', such as 2.045).

Examples of number literals: 

typeof 25; // "number"
typeof 2.045; // "number"

let age = 28;
console.log(age); // "number"

As demonstrated above, JavaScript doesn't distinguish between integers and floating point decimals. Although in ES6, there's a handy method called Number.isInteger() that can be used to check if a number is an integer. 

Number.isInteger(42); // true
Number.isInteger(3.142); // false

Just like strings, numbers also have a constructor function: 

new Number(3); // 3

Octal and Hexadecimal Numbers

Numbers that start with 0x are considered to be in hexadecimal (base 16) notation. In ES6 there's also support for Octal literals (base 8) if a number starts with 0o (the number zero followed by the letter "o") and binary literals if a number starts with 0b: 

console.log(0xAF); // 175
console.log(0o47); //39
console.log(0b1010); //10

Exponential Notation

Numbers can be represented in exponential notation, which is shorthand for "multiply by 10 to the power of" 

1e6; // means 1 multiplied by 10 to the power 6 (a million)
<< 1000000
2E3; // 2 multiplied by 10^3 (two thousand)
<< 2000

2.5e-3; // means 2.5 multiplied by 10 to the power -3 (0.001)
<< 0.0025

5 .toExponential(); //Or
